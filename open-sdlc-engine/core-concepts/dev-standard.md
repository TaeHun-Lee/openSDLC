# 개발 표준 (Dev Standards)

OpenSDLC v1 프로젝트 수행 시 준수해야 할 물리적 구조와 기술 표준입니다.

## 1. 아티팩트 관리 구조
모든 결과물은 `workspace` 폴더 하위에 프로젝트 단위로 관리됩니다.

- **프로젝트 루트**: `workspace/{ProjectName}/`
- **이터레이션 폴더**: `workspace/{ProjectName}/iteration-{NN}/`
- **아티팩트 저장소**: `workspace/{ProjectName}/iteration-{NN}/artifacts/`
- **검증 보고서**: `workspace/{ProjectName}/iteration-{NN}/verification_report.md` (PMAgent가 생성)

## 2. 코드 구조 및 생성 규칙
- **개발 환경**: 대상 프로젝트의 목적과 사용자가 지정한 기술 스택, 런타임, 실행 방식에 맞춰 개발합니다. 특별한 제약이나 별도 지정이 없다면 가장 단순하고 재현 가능한 실행 환경을 우선합니다.
- **소스 위치**: 최신 구현 결과는 기본적으로 `workspace/{ProjectName}/dev/` 하위에 유지합니다. 단, 하위 디렉터리 구조와 실행 엔트리포인트는 프로젝트 유형(웹, CLI, 서버, 라이브러리 등)에 따라 달라질 수 있습니다.
- **지속성**: 새로운 이터레이션에서 코드가 수정될 때마다 `dev` 폴더의 파일들이 최신 상태를 유지하도록 업데이트합니다.

## 3. 에이전트 운영 표준
- **투명성**: PMAgent는 에이전트 전환과 순서를 통제하며, 실제 작업 시작/진행/완료 보고는 현재 활성 에이전트가 자신의 1인칭 시점으로 직접 수행합니다.
- **보고**: 각 에이전트는 작업이 완료되면 결과 아티팩트의 링크와 요약 내용을 포함하여 자신의 진행 상황을 직접 보고합니다.
- **보고 시점/시점 규칙**: 진행 보고는 3인칭 설명이 아니라 현재 작업 중인 Agent 자신의 1인칭 기준으로 작성합니다.
- **보고 형식 규칙**: 각 Agent는 아래 형식을 기본 패턴으로 사용합니다.
  - `[XXXAgent] 지금부터 OOO 작업을 시작합니다. 주요 작업은 아래와 같습니다.`
  - `[] 세부작업1`
  - `[] 세부작업2`
  - `[XXXAgent] 세부작업들에 대한 진행 내용들...`
  - `[XXXAgent] 작업 완료 보고와 다음 작업을 진행할 대상 Agent 정보`
- **엄격성**: 사용자가 명시하지 않은 스타일 변경, 기능 추가는 지양하며 사용자가 지정한 기술 스택과 런타임을 우선적으로 따릅니다.

## 4. 검증 운영 표준
- **Validator 우선 원칙**: 에이전트의 자기 점검 결과만으로 산출물을 승인하지 않으며, 가능한 한 기계적으로 재검증 가능한 규칙을 우선 적용합니다.
- **스키마 검증 의무**: 모든 YAML 아티팩트는 필수 필드, 필드 타입, 허용 상태값, `artifact_type`-`created_by` 조합을 검증해야 합니다.
- **추적성 검증 의무**: `UC -> TEST-DESIGN -> IMPL -> TEST-EXECUTION -> FB` 참조 체인은 끊기면 안 되며, `source_artifact_ids`, `traceability`, `test_scope`, `related_defect_id`는 실제 상위 아티팩트와 일치해야 합니다.
- **증거 검증 의무**: `TestReportArtifact`의 각 실패 항목은 비어 있지 않은 `evidence`를 포함해야 하며, PMAgent는 최소 일부 증거를 실제 파일, 경로, 로그, 문자열과 대조해야 합니다.
- **판정 일관성 검증**: `satisfaction_score`, defect 심각도, `FeedbackArtifact`의 상태와 결정은 서로 모순되면 안 됩니다. 예를 들어 점수가 90 미만이면 다음 이터레이션이 필요하며, 고위험 결함이 남아 있으면 자동 완료를 권고할 수 없습니다.
- **역할 경계 보호**: Validator는 형식 검사를 넘어서 역할 침범도 감시해야 합니다. 예를 들어 CodeAgent가 요구사항 범위를 임의 확장하거나 CoordAgent가 defect와 무관한 새 요구를 추가하면 경고 또는 실패로 처리합니다.
- **승인 게이트**: PMAgent는 validator 통과 결과 없이 `verification_report.md`에서 준수 완료를 선언할 수 없습니다.
